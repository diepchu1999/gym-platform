# Status Flow Catalog

## Member Status

```text
LEAD -> REGISTERED -> KYC_PENDING -> ACTIVE
ACTIVE -> INACTIVE
ACTIVE -> SUSPENDED
ACTIVE -> BLACKLISTED
```

Meanings:
- LEAD: potential customer, not yet registered as full member.
- REGISTERED: account/profile created.
- KYC_PENDING: waiting for CCCD/student verification.
- ACTIVE: valid member.
- INACTIVE: no active package or inactive for a long time.
- SUSPENDED: temporarily blocked by system/admin rule.
- BLACKLISTED: blocked due to serious issue.

## KYC / Verification Status

NOT_SUBMITTED -> PENDING -> APPROVED
PENDING -> REJECTED
PENDING -> REQUEST_RESUBMIT
APPROVED -> EXPIRED
```

```

## Installment Application Status

```text
DRAFT -> SUBMITTED -> PENDING_PROVIDER_APPROVAL -> APPROVED -> DISBURSED
PENDING_PROVIDER_APPROVAL -> REJECTED
DRAFT -> CANCELLED
SUBMITTED -> CANCELLED
```

## Booking Status

```text
DRAFT -> PENDING_PAYMENT -> CONFIRMED -> CHECKED_IN -> IN_PROGRESS -> COMPLETED
CONFIRMED -> WAITING_CUSTOMER_CONFIRMATION -> CHECKED_IN
WAITING_CUSTOMER_CONFIRMATION -> NO_SHOW
PENDING_PAYMENT -> EXPIRED
CONFIRMED -> CANCELLED
CONFIRMED -> NO_SHOW
PAID_BOOKING -> REFUNDED
```

Notes:
- Free/quota bookings can skip PENDING_PAYMENT and go directly to CONFIRMED.
- Paid bookings should start as PENDING_PAYMENT and expire if not paid within hold time.
